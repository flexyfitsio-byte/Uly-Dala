# Uly Dala — MVP

Цифровой ИИ-гид по Казахстану: карта мест, ИИ-гид с генерацией маршрутов (RAG,
без «галлюцинаций»), геймификация (XP, уровни, викторины) и B2B-кабинет для партнёров.

## Быстрый старт

```bash
cd uly_dala_startup
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env             # затем при желании впишите OPENAI_API_KEY
python app.py
```

Сайт откроется на `http://localhost:5000`.

**Важно:** MVP работает и без `OPENAI_API_KEY` — ИИ-гид и ИИ-историк переключаются
в демо-режим и собирают ответы по простым правилам на основе тех же данных.
Это сделано специально, чтобы можно было тестировать интерфейс без затрат на API.

## Что уже работает

- **Главная** (`/`) — карточки мест с фильтрами, сетка из 6 сервисов, XP-профиль.
- **ИИ-гид** (`/chat`) — настоящий диалог с памятью: генерирует маршрут и умеет
  дорабатывать его по уточнениям («сделай дешевле», «добавь красивые места»,
  «сократи маршрут»). RAG: модель использует только места из `data/places.json`.
- **История и культура** — клик по месту открывает карточку с историей и
  мини-викториной; XP начисляется один раз за действие (без накрутки кликами).
- **Туры / Отели / Аренда авто** (`/tours`, `/hotels`, `/cars`) — витрины с
  демо-каталогом и формой заявки на бронирование (лид для менеджера, без оплаты).
- **Авиа/ЖД билеты** (`/flights`) — форма заявки на подбор рейса (без каталога —
  реальные расписания требуют партнёрского API, см. ниже).
- **Регистрация/вход** (`/register`, `/login`) — простая сессия, пароли хэшированы.
- **Партнёрский кабинет** (`/partner`) — заявка на добавление места в базу.

## Что нарочно упрощено (для следующих итераций)

- **Данные — в JSON-файлах**, а не в БД. `database/db_manager.py` спроектирован
  так, чтобы позже заменить реализацию на PostgreSQL, не трогая роуты.
- **Туры/Отели/Авто/Билеты — демо-каталог и заявки, а не реальное бронирование.**
  Кнопка «Забронировать» сохраняет лид (`data/booking_requests.json`), деньги не
  списываются. Настоящая интеграция требует партнёрского API:
  - Авиа/ЖД: Aviasales/Kiwi, Chocotravel, Aviata
  - Отели: Booking.com Affiliate, Ostrovok
  - Авто: локальный агрегатор проката
  Как только появятся API-ключи от партнёров — эти роуты (`routes/services.py`)
  нужно заменить на реальные вызовы вместо `data/tours.json` и т.п.
- **B2B API без JWT** — заявки партнёров не изолированы и доступны без токена.
- **Карта** — сейчас список карточек; `lat`/`lng` в данных уже есть, интерактивную
  карту (Leaflet/Mapbox) можно добавить без изменений бэкенда.
- **ИИ-переводчик, сезонные события, коллекции, дневник путешествий** из полной
  концепции пока не реализованы — структура проекта готова их принять.

## Структура

```text
uly_dala_startup/
├── app.py                  # точка входа
├── config.py                # конфигурация (.env)
├── data/                    # JSON вместо БД на старте
│   ├── places.json
│   ├── users.json           # профили геймификации (создаётся автоматически)
│   └── auth_users.json      # логины/хэши паролей (создаётся автоматически)
├── ai_engine/                # ИИ-гид (RAG + JSON-mode + демо-режим без ключа)
├── database/                 # доступ к данным (JSON, легко заменить на БД)
├── routes/                   # блюпринты Flask (main, auth, ai, gamification, partner)
├── static/{css,js}
└── templates/                # HTML в фирменном стиле (тёмно-синий + золото)
```

## Деплой на Railway

Файл `Procfile` уже готов (`gunicorn app:app`). Достаточно подключить репозиторий
к Railway и задать переменные окружения `SECRET_KEY` и (опционально) `OPENAI_API_KEY`.

## Следующие шаги (предложение)

1. Подключить интерактивную карту (Leaflet) на главной — данные `lat/lng` уже есть.
2. Перейти на PostgreSQL, когда понадобится реальная персистентность и партнёрские кабинеты.
3. Добавить JWT для `routes/partner_api.py` перед первыми реальными партнёрами.
4. Добавить ИИ-историка на страницу карточки места (эндпоинт `/api/ai/historian/<id>` уже готов).
